# ORBIT — プロダクト仕様(プロトタイプ準拠)

この文書は、HTML プロトタイプ `docs/prototype/orbit_space_wallpaper_app.html` から抽出した、Flutter 本実装の対象仕様。プロトタイプと本仕様が食い違う場合はプロトタイプを確認して本仕様を更新する。

## コンセプト

NASA の宇宙写真を「待ち受け」風に眺めるアプリ「ORBIT」。観測機器の HUD(時計・座標・レチクル・写真情報)を重ね、スワイプで写真を切り替え、タップで HUD を消して没入できる。カスタマイズ・お気に入り・待ち受けプレビューを持つ。

**毎回ちがう写真**を見せるのが体験の核。これを「端末ローカルの写真プールからランダムに 1 枚選ぶ」方式で実現する(下記「写真プールと補充」)。

## 写真プールと補充(本実装の中心方針)

プロトタイプは起動のたびに NASA API を叩くが、**本実装では通信を表示経路から外す**。

- 端末に数十〜数百枚の写真(画像バイナリ + メタデータ)を**ローカルにキャッシュ(プール)**しておく。
- **起動時はプールからランダムに 1 枚選ぶだけ**。ここでネットを一切叩かない。これにより:
  - **オフラインでも動く**(圏外・機内モードで真っ白を避ける。待ち受けアプリでは重要)。
  - **起動が速い**(通信待ちが発生しない)。
  - **レートリミットと無縁**。
- **プールの補充は別タイミングで低頻度に行う**。「前回更新から 24 時間以上経っていたら、起動後にバックグラウンドで何枚か取得してプールを入れ替える」。ユーザー体感は毎回新鮮なのに、実際の通信は 1 日 1 回程度に圧縮される。
- 補充に失敗しても(オフライン・API 不調)、**既存プールは保持**してアプリは必ず起動・表示できる。
- 選択・補充対象の乱数は `Random(seed)` を注入し、失敗再現のため seed を残す。

詳細な決定は ADR 参照: プール方式 [0001](design-docs/0001-photo-pool-cache.md)、写真供給元 [0003](design-docs/0003-photo-source.md)、オフライン・フォールバック [0004](design-docs/0004-offline-fallback.md)。

## 画面構成

1 画面(ビューア) + 2 ボトムシート + 1 モーダル + オーバーレイ群。

1. **ビューア**(写真 + HUD + ドック)
2. **カスタマイズシート**(TUNE)
3. **コレクションシート**(SAVED)
4. **待ち受けプレビューモーダル**(WALLPAPER)
5. **トースト** / **ローダー**

### ビューア

- **写真レイヤ**: プールから選ばれた写真を全画面表示。クロスフェード切替。可読性のためのビネット(上下グラデーション)。
- **HUD レイヤ**(タップで表示/非表示トグル):
  - ステータスバー(時刻、信号アイコン、"ORBIT")。
  - 四隅のコーナーブラケット。
  - 中央レチクル(照準)。
  - 左テレメトリ(RA / DEC / TRACKING)、右テレメトリ(フレーム番号 / MAG / EXP)。座標はインデックスから導出する擬似値(演出)。
  - 大きい時計(位置: 上 / 中央 / 下、サイズ: S / M / L、24h 切替)+ 日付。
  - 写真メタ(カテゴリ eyebrow / タイトル / "NASA · center" / 日付)。
  - 進捗ドット(最大 8 個)。
  - タップヒント(初回操作後に消える)。
- **ドック**: SAVE(お気に入りトグル)、SAVED(コレクションを開く)、WALLPAPER(待ち受けプレビュー)、TUNE(カスタマイズを開く)。
- **再取得スキャン演出**: 写真切替時にスキャンラインが走る("RE-ACQUIRING TARGET")。

### カスタマイズシート(TUNE)

- **観測テーマ**: カテゴリチップ(複数定義から 1 つ選択)。選択でそのテーマのプール表示に切り替える。
  - カテゴリ: 星雲 / 銀河 / 地球 / 火星 / 月 / 木星 / 太陽 / 深宇宙 / オーロラ / 土星(プロトタイプの `CATEGORIES` 準拠)。
- **時計**: 表示 ON/OFF、位置(上/中央/下)、サイズ(S/M/L)、24 時間表示 ON/OFF。
- **HUD(観測機器の表示)**: 座標テレメトリ ON/OFF、レチクル ON/OFF、写真情報 ON/OFF。
- **アンビエント**: 自動スライド ON/OFF、切替間隔(6s/12s/30s)。

### コレクションシート(SAVED)

- 保存した写真のサムネイルグリッド(3 列)。タップでその写真をビューアに表示。
- 空のときは案内文を表示。

### 待ち受けプレビューモーダル(WALLPAPER)

- 現在の写真をロック画面風(日付 + 時刻 + フッタ)にプレビュー。
- 「この宇宙を待ち受けにする」(実機では写真保存 → OS の壁紙設定へ誘導)、「閉じる」。

## 操作

- **スワイプ左右**: 前 / 次の写真。
- **タップ**(移動なし・短時間): HUD の表示 / 非表示トグル。
- **キーボード**(デバッグ / Web): ← → ↑ ↓ で前後、スペースで HUD トグル。
- **自動スライド ON 時**: 設定間隔で自動的に次へ。

## 状態・データ

### 写真

- `Photo`: `{ id, title, center(NASA センター名), date, category, imageRef(ローカルキャッシュ参照), sourceUrl }`。
- `Pool`: 写真リスト + `lastRefreshedAt`(最終補充時刻)+ カテゴリ別の構成。
- お気に入り(`favorites`): 保存した写真の集合(ローカル永続化)。

### 設定

プロトタイプの `state.settings` 相当(すべてローカル永続化):

- 時計: `clock`(表示)/ `clockPos`(top/center/bottom)/ `clockSize`(s/m/l)/ `h24`。
- HUD: `tel`(テレメトリ)/ `reticle` / `meta`(写真情報)。
- アンビエント: `auto`(自動スライド)/ `interval`(6/12/30 秒)。
- 観測テーマ: 現在のカテゴリ。

プロトタイプはメモリ保持のみ(リロードで消える)。**本実装では設定・お気に入り・プールをローカル永続化する**(差し替え可能な `SettingsStore` / `CollectionStore` / `PoolStore` 境界の対象)。

### データ不変条件(テストで保証する)

- プールの各 `Photo.id` は一意。
- カテゴリは定義済みの値のみ。
- 補充後もプール件数は下限を割らない(補充失敗時は旧プール維持)。
- `lastRefreshedAt` と現在時刻の差が 24h 未満なら補充をスキップする。

## デザイン

- デザイントークンはプロトタイプの `:root` 定義が正(void `#060912`、HUD `#E8ECF2`、amber `#F5A623`、muted `#6E7891` ほか)。常時ダーク基調。
- フォント: Space Grotesk(見出し)、Space Mono(HUD / 数字)、Noto Sans JP(本文)。
- アニメーション: クロスフェード、スキャンライン、シート / モーダルのスライド。`prefers-reduced-motion`(Flutter では `MediaQuery.disableAnimations`)で抑制する。
- アクセシビリティ: ドックボタン・トグル・チップ等に意味ラベル(Flutter では Semantics)、focus 可視化。

## スコープ外(初期)

- バックエンド・認証・課金(NASA Images API はキー不要)。
- 複数 locale(日本語のみ)。
- push notification / OS のバックグラウンド実行枠での補充(補充は起動後フォアグラウンドで実行)。
- 実機でのロック画面壁紙の自動適用(写真保存 + OS 設定への誘導にとどめる。iOS の制約が強い)。
